# タモツ MVP ロードマップ

> 直近 8 週間で iPhone / Mac 両対応の最低限プロトタイプを完成させ、実際の業務に投入するための計画です。

## ウィークリーミッション
- **Week 1**: Xcode プロジェクト初期化、CloudKit コンテナ作成、`Task` モデルとローカル保存（SwiftData）。
- **Week 2**: タスク一覧 UI（Today / Upcoming）とタスク追加フローの基本 UI 完成。
- **Week 3**: 「小ゴール」入力とバリデーション、タスク編集/完了フロー実装。
- **Week 4**: やれない理由の記録画面、原因タグの集計ロジックを追加。
- **Week 5**: 日次チェックイン（朝・昼・夜）UI とローカル通知の設定。
- **Week 6**: 週次レビュー画面の初版、Mood/Reason ダッシュボード整備。
- **Week 7**: 同期レイヤー（CloudKit or Supabase）を安定化、競合解決ロジックの整備。
- **Week 8**: ベータ運用（2 週間）のための改善、バグ修正、初期データの移行スクリプト作成。

## ガイドライン
- 週末に GitHub Issue を棚卸しし、翌週のフォーカスを 3 つまでに絞る。
- 毎日終了時に `docs/journal/YYYY-MM-DD.md` にログを残す（後日追加予定）。
- モチベーションが落ちた日も「やれなかった理由」だけは 1 行で記録。

## 依存関係
- Apple Developer Program への加入 → CloudKit/通知利用。
- iCloud 同期を採用する場合、開発者証明書と App ID の準備。
- Supabase を採用する場合、プロジェクト作成と RLS 設計が必要。

## リスクと対策
- **継続利用できない UI**: 1 週間ごとに実際のタスクを入力して痛点を洗い出す。改善点は即 Issue 化。
- **同期トラブル**: 初期は 1 デバイスで検証。同期は日報を取りながら少しずつ導入。
- **感情ログが負荷になる**: チェックインは 30 秒完了を目指し、質問数を極力絞る。必要以上の入力を強制しない。
